This section outlines how to set up your OutdoorNav system for initial testing.

- Ensure that the UGV is powered on.
- If the optional Base Station is included, ensure that it is also powered on.


## Networking Setup and UI Access {#networking_setup_and_ui_access}

It is possible to use either a Windows or a Linux PC to connect to the robot and OutdoorNav user interface (UI).
There are a few different ways to connect/communicate with the robot, each have their pros/cons.
Below, each method is listed with instructions on how to access the UGV.

<details>
<summary>**Via Ethernet**</summary>

### Via Ethernet

Connecting to the robot via the ethernet port allows for the fastest connection speed, ideal for
transferring large files and for quick/easy access to the robot. It is not possible to
connect via ethernet if you wish to run any Outdoornav missions.

1. Connect an ethernet cable between your PC (Windows/Linux) and the ethernet port on the rear of the robot.
2. Set up the ethernet port on your PC to have a static IP address on the 192.168.131.XXX
subnet (eg. 192.168.131.200)
3. Open a terminal (Linux)/command prompt (Windows) and test the communication between your PC and the robot PC

      ```code
    ping 192.168.131.1
    ```

4. If the ping test is successfull and you are receiving replies, then you can SSH into the robot's PC 
and access the UI at the following,

    |                     |                                       |
    | ------------------- | ------------------------------------  |
    | SSH (password)      | `ssh robot@192.168.131.1` (clearpath) |
    | UI (URL)            | http://192.168.131.1                  |

    If the ping test was unsuccessful, then confirm that you have set up your ethernet interface
    properly to have a static IP on the correct subnet 192.168.131.XXX (eg. 192.168.131.200)

      ```code
    ifconfig (Linux)
    ipconfig (Windows)
    ```

    If you are still having issues, please contact [Support](../support.mdx).

#### Access the Robot Peplink

|               |                       |
| ------------- | --------------------- |
| Peplink (URL) | http://192.168.131.51 |
| Username      | admin                 |
| Password      | Clearpath123          |


:::note

The robot PC will not have internet access unless you have connected the onboard robot Peplink to an
internet source (ie. connecting via wifi or by adding a SIM card and connecting to a carriers network)

To connect the robot Peplink to a internet source follow [these instructions](#robot_peplink_connect_to_internet).

:::

</details>

<summary>**Via a CPR provided Base Station**</summary>

### Via a CPR provided Base Station

If you have purchased a seperate (optional) Base Station, it will be configured to act as the 
a wireless access point for your robot. 

  - Place the base station outside with a clear view of the sky. It should be at least 5m away
  from large buildings or trees.
  - Power on the base station.
  - The base station will automatically connect to the robot and provide a wifi network for you
    to connect to.
  - The base station will also automatically start an auto-survey and will start providing an
    RTK fix to the robot in approx. 10 - 15 minutes.

:::note

If you move the base station while it is still powered on, it will need to be powered off and
back on again to re-survey and provide an accurate RTK fix.

:::

<details>
<summary>**Via Cellular**</summary>

### Via Cellular

Connecting via cellular is likely the most versatile method for deployment of your robot. It allows
the robot to access the internet to receive RTK corrections and also allows for remote access, almost
anywhere in the world.

Requirements:

1. SIM card with a data plan (prepaid or plan) from a local network provider.
2. A tunnelling software (eg. Cloudflare)

</details>


### Getting Internet Access {#robot_peplink_connect_to_internet}

#### Connect Robot Peplink to Wifi

The first option is to connect the robot Peplink radio (client) to a wifi source.

1. Access the robot Peplink via one of the methods above

<center>
  <figure>
    <img src="/img/outdoornav_images/robot_peplink_dashboard.png" />
    <figcaption>Robot Peplink Dashboard</figcaption>
  </figure>
</center>

2. Depending on the type of Wifi network you are connecting to (2.4 or 5 GHz), you will click on
the appropriate `Wi-Fi WAN on X GHz` connection in the list of connections.
3. Toggle the `Enable` field.
4. Scroll to the bottom of the `WAN Connection Settings` to the `Wi-Fi Connection Profiles` section.
5. Click `Create Profile...`

<center>
  <figure>
    <img src="/img/outdoornav_images/robot_peplink_wifi_profile.png" />
    <figcaption>Robot Peplink Wifi Profile</figcaption>
  </figure>
</center>

6. Fill in all the required fields for the wifi network you are connecting to. Click `OK`.
7. Click `Save and Apply` at the bottom right of the `WAN Connection Settings` pop-up.
8. Click `Apply Changes` at the top right of the dashboard page. The peplink should be trying to
connect to the SSID of the profile you created.

#### Add a SIM card to the Robot Peplink

Use an existing SIM card or purchase a SIM card to be used for internet access to the robot.

1. Insert the SIM card into Slot A or B on the peplink (under screwed cover labeled `Cellular SIM`)

<center>
  <figure>
    <img src="/img/accessories_images/peplink_radio_1.png" />
    <figcaption>Peplink Max BR1 Pro 5G Cellular Router</figcaption>
  </figure>
</center>

2. Access the robot Peplink via one of the methods above.
3. Click on the `Cellular` connection in the list of connections on the dashboard.
4. Ensure that the `Enable` field is toggled to ON.
5. Click `Save and Apply` at the bottom right of the `WAN Connection Settings` pop-up.
6. Drag and dro whatever connection is in the priority 1 slot and drag and drop the cellular connection
to priority 1.
7. If you have added a data plan enabled on your SIM card, then it should automatically
   connect to the carriers network and provide an IP in the connections list.
8. If you click the IP address it will pop-up all of the details of the cellular SIM.

#### Connect Base Station to Internet

There are two different ways to connect the base station to the internet, either via wifi or by adding a SIM
card to the Base Station Peplink. They are listed below.

1. You can insert a SIM card into the Base stations Peplink in a similar manner to the section above.
  
2. You can connect the `WAN` ethernet port on the side of the base station to a live internet
connection (physical ethernet cable) and the Base Station should be granted an IP address via DHCP.  If the Base Station is
not able to connect to the Internet, or is not granted a WAN IP address, contact your network
administrator in case additional login credentials are needed.

## RTK Corrections Setup {#rtk_setup}

RTK corrections are required in order for the XVN to output a localization. OutdoorNav will not work
if you do not have RTK corrections set up.

1. Open the OutdoorNav UI. Click one of the GNSS status indicators and then click the `Open XVN UI` button.
2. In the XVN UI, navigate to Status (top bar) -> Fusion. Click the `Stop` button to stop the fusion.
3. Navigate to Configuration (top bar) -> GNSS

<center>
  <figure>
    <img src="/img/outdoornav_images/xvn_rtk_setup.png" />
    <figcaption>XVN RTK Setup Page</figcaption>
  </figure>
</center>

  - Enter the details below depending on your connectivity source.
  - Click `Save & Apply`.
4. Confirm connection to RTK correction server.
  - The `Correction data` indicator will turn green and say `connected`.
  - You should also check the Status -> GNSS page and confirm that `good/complete` corrections have been
  comming in for at least one minute.
5. Restart the fusion by navigating to Status (top bar) -> Fusion and clicking the `Start` button.
  You should start seeing the indicators on the page start turning green as the fusion starts converging.

<center>
  <figure>
    <img src="/img/outdoornav_images/xvn_fusion_start.png" />
    <figcaption>XVN Fusion Start</figcaption>

## Check the Localization Status GPS RTK Fix {#checking-localization-status}

The POS (position), DIR (heading), and FUS (sensor fusion) GNSS status indicators on the UI
should be green: <img src="/img/outdoornav_images/ui_status_indicators-2_0_0.png" width="75px" />.

:::note

If the GNSS status indicators are yellow or red, the selected location
does not have an adequate GPS signal. Move the UGV such that the GNSS
antennas have clearer visibility to satellites.

:::

Click any of the GNSS status indicator buttons or Navigate to
**Menu** &srarr; **TOOLS** &srarr; **Status Monitor** to display the system status. All 5 localization
statuses should be green. If the `IMU Status` and/or the `Wheelspeed` statuses are not green they should
turn green once the robot drives around a little while, until the two IMU and wheel encoder data have
converged.

## Complete the System Configuration {#system_config}

### Load Map Tiles {#loading_map_tiles}

The Web UI does not cache map tiles on the browser; therefore, the user
will need to ensure their computer has a connection to an Internet
source to load the map tiles. There are a several options to achieve
this:

1.  If the system includes a Base Station that is connected to the
    Internet, you will be able to access the Internet and therefore map
    tiles over the base station network without needing any further
    updates.
2.  Connect your phone via USB to your computer (or tablet) and enable
    USB tethering on your phone to share the Internet from your phone.
3.  Switch your computer (or tablet) to a network that is connected to
    the Internet, zoom in/out of the map to load the tiles then switch
    back to the original network.

### Set Datum

Navigate to **Menu** &srarr; **Settings**. In the `Map Settings` section, enter the latitude
and longitude of the datum you want to set. The datum will represent the (x,y) = (0,0) map coordinate frame
origin and must be set within 1 km of your deployment location.

### Set Dock Location

:::note

If the autonomous docking package has been included,
follow these instructions to set up the dock location. Otherwise, this
section can be skipped.

:::

:::warning

Keep the area around the dock free of objects and people. There is no
obstacle detection between the pre-docking point and the dock;
similarly, there is no obstacle detection during the undocking
operation.

:::

<center>
  <figure>
    <img width="200px" src="/img/outdoornav_images/a300_dock_target.png" />
    <figcaption>Husky A300 Autocharge Dock</figcaption>
  </figure>
  <figure>
    <img width="400px" src="/img/outdoornav_images/a300_charging.png" />
    <figcaption>Husky A300 at Dock</figcaption>
  </figure>
</center>

Before being able to dock the UGV autonomously, set up the dock
location. Follow these steps to position the UGV in its correct position
and orientation.

1.  Power ON the UGV and wait for the system to finish booting.

2.  Start by manually driving the UGV to the dock target and align it as
    straight and centered as possible so that it begins charging. 

3.  On the left side of the UI, select the Edit mode button (pencil icon) and then select the Docks option.

4.  Click anywhere on the map to create a dock. Name it and select the default configuration.

5.  Click and drag the dock to the location of the robot icon and align its position and orientation.
    To rotate the dock, hold the `Ctrl` button and use the cursor to orient the dock (the arrow on the
    dock should be used as a guide).

6.  On the left side of the UI, select the Edit mode button and then select the cursor option.

7.  With the remote controller, drive the robot at low speed purely in reverse so the robot icon on the 
    UI lines up with the green dot on the UI (ie. the predock point).

8.  Right click the dock and select `Survey` in the list of option.

9.  The docks position/orientation will be updated in 10 seconds. You should see the dock snap to its new pose.

## Create Your First Map

1.  Ensure that the UI is in "Map Edit Mode":
    <img src="/img/outdoornav_images/ui_edit_button.png" width="40px" />  &srarr; 
    <img src="/img/outdoornav_images/ui_edit_map.png" width="40px" />.
2.  Open the drop down menu in the bottom bar (left-side) and select the "Add Map" option.
3.  Start placing points and building paths. (A map will be created with the default settings.)

For more details, refer to [Map Creation](../web_user_interface/ui_map_mode#mapCreation).

## Create Your First Mission

1.  Ensure that the UI is in "Mission Edit Mode":
    <img src="/img/outdoornav_images/ui_edit_button.png" width="40px" />  &srarr; 
    <img src="/img/outdoornav_images/ui_edit_mission.png" width="40px" />.
2.  Open the drop down menu in the bottom bar (right side) and select the "Add Mission" option.
3.  Select the "Goalpoint Mode" button.
4.  Click on the map to add a goalpoint.
5.  Confirm that a preview path is shown on the map.

For more details, refer to [Mission Creation](../web_user_interface/ui_map_mode#missionCreation).

## Execute a GoTo

1.  Right-click on a location within the map (ie. anywhere that is in the green navigable space).
2.  Select `GoTo`, a previewed path will show up. If you approve of the path that the robot will
traverse, click the checkmark. The robot will begin navigating to the selected point.


## Execute Your First Mission

1.  In the bottom bar of the UI, click the `Play` button <img src="/img/outdoornav_images/ui_start_button_simple.png" width="16px"/>.
2.  Confirm that the "Play" button has turned green, which indicates that the mission has started.

For more details, refer to [Mission Execution](../web_user_interface/ui_map_mode#missionExecution).
