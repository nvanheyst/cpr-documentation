#### Package and Setup
Luxonis OAK-D cameras use the `depthai_ros_driver` ROS 2 driver. The driver is open-source,
maintained by Luxonis, and hosted on [GitHub](https://github.com/luxonis/depthai-ros.git). This
package and its dependencies are installed alongside the `clearpath_sensors` package.

For specifics on the way Clearpath's configuration system launches the camera, see the Luxonis OAK-D
[launch file](https://github.com/clearpathrobotics/clearpath_robot/blob/jazzy/clearpath_sensors/launch/luxonis_oakd.launch.py)
and the [default parameter file](https://github.com/clearpathrobotics/clearpath_robot/blob/jazzy/clearpath_sensors/config/luxonis_oakd.yaml)
in `clearpath_sensors`.

#### Device Type

The `device_type` parameter specified what specific model of OAK-D camera is used. Supported types are:
- `pro`: the [OAK-D Pro](https://shop.luxonis.com/products/oak-d-pro) (default);
- `lite`: the [OAK-D Lite](https://shop.luxonis.com/products/oak-d-lite-1); and
- `pro_w_poe`: the [OAK-D Pro W PoE](https://shop.luxonis.com/products/oak-d-pro-w-poe)

#### Multiple Cameras

If you have multiple OAK-D cameras configured, you must specify either the `camera.i_mx_id`
(for USB cameras) or the `camera.i_ip` parameter (for ethernet/PoE cameras).

See [Luxonis' documentation](https://docs.luxonis.com/software/depthai/examples/device_information)
for instructions on how to determine your camera's IP address or MxId.
