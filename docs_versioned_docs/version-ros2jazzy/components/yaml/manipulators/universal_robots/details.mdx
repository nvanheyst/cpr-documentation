### Package and Setup {#ur_package_and_setup}
The Universal Robots arms use the `ur_description` and `ur_driver` ROS 2 Packages. The description and driver are open source, maintained by Universal Robots, and are hosted on GitHub. Refer to the [description repository](https://github.com/UniversalRobots/Universal_Robots_ROS2_Description) and [driver repository](https://github.com/UniversalRobots/Universal_Robots_ROS2_Driver).

For more specifics on the way Clearpath's configuration system adds the arm to the robot description, see the [description file](https://github.com/clearpathrobotics/clearpath_common/blob/jazzy/clearpath_manipulators_description/urdf/arm/universal_robots.urdf.xacro). Note, all parameters to the `xacro:macro universal_robots` can be passed through the `robot.yaml` entry above. For examples, read the sections below.

For more information on how to setup the arm, see the [Universal Robots ROS 2 Driver Documentation](https://docs.universal-robots.com/Universal_Robots_ROS2_Documentation/index.html).

### Standard Initialization (Recommended) {#ur_standard_initialization}
At start up, the robot computer will launch the arm's controller using the `clearpath-manipulators.service`. Once the arm is on and initialized through the teach pendant, start the `external_control` program, which will begin communication with the ROS driver.

Start the program by pressing the play button on the **Run** page.

### Headless Initialization {#ur_headless_initialization}
If the arm has been setup to accept external control by surrendering manual control in the teach pendant, then it is possible to bypass the manual initialization of the UR program on the tablet. If this is the case, make sure the arm has been turned on and initialized through the teach pendant, then set, `headless: true` in the `robot.yaml`. This will restart the services and the arm should connect automatically.

### Parameter: Device IP {#ur_param_ip}
The `robot_ip` must be set to match the networking settings of the arm's control box. By default, we use the `192.168.131.40` address.

### Parameter: UR Type {#ur_param_type}
The `ur_type` parameter can be modified to change the type of the arm. By default, it is set to `ur5e`. But, it is critical that it is changed to match the real arm type. The following types are supported: `ur3`, `ur3e`, `ur5`, `ur5e`, `ur10`, `ur10e`, `ur16e`, `ur20`, `ur30`.

### Parameter: Calibration Parameters {#ur_param_calib}
By default, the default kinematics parameter file is passed to the arm's ROS controller. Because each arm has it's own measurable offsets, it is likely that without updating this calibration file the arm's movements will not be as accurate as could be. Therefore, use the [`ur_calibration`](https://docs.universal-robots.com/Universal_Robots_ROS2_Documentation/doc/ur_robot_driver/ur_robot_driver/doc/installation/robot_setup.html#extract-calibration-information) package to obtain the calibration parameter file.

<details>

<summary>UR Retrieve Calibration</summary>

```bash
ros2 launch ur_calibration calibration_correction.launch.py robot_ip:=<robot_ip> target_filename:="${HOME}/my_robot_calibration.yaml"
```

Then, in the entry to the configuration file, pass in the path to the calibrated kinematics file.
```yaml
manipulators:
  moveit:
    enable: false
  arms:
    - model: universal_robots
      parent: default_mount
      xyz: [0.0, 0.0, 0.0]
      rpy: [0.0, 0.0, 0.0]
      ur_type: ur5e # ur3, ur3e, ur5, ur5e, ur10, ur10e, ur16e, ur20, ur30
      robot_ip: 192.168.131.40
      headless: false
      kinematics_parameters_file: "/path/to/my_robot_calibration.yaml"
```
Note, you can also pass in substitution arguments:
```yaml
      kinematics_parameters_file: "$(find package_name)/path/to/my_robot_calibration.yaml"
```

</details>


### Robotiq Gripper
On the UR arms, the Robotiq gripper is exposed to the robot computer as a serial device.

It is possible to connect the gripper to the robot computer directly using a serial (RS485) to USB cable and externally routing the cable down the arm using anchors to prevent the cable from interfering with the arm's motion.

However, it is recommended to use a UR coupling kit attached at the wrist to connect the UR directly to the arm's wrist interface.

#### URCap for Wrist Adapter
If the gripper has been attached to the arm using the appropriate wrist adapter, we can install a URCap to command the gripper through the arm.

Using the same method as discussed in the [Teach Pendant Setup/URCap](#ur_urcap), install the [Universal_Robots_ToolComm_Forwarder_URCap](https://github.com/UniversalRobots/Universal_Robots_ToolComm_Forwarder_URCap).

The `RS485 URCap` installs a daemon on the UR controller that executes a script to setup a `socat` connection to expose the tool's serial device by forwarding the port to a remote computer.
Once installed, if the Universal Robot configuration in the `robot.yaml` contains the `gripper.model.robotiq_2f_85` entry, then the `RobotiqHardwareInterface` will be launched alongside the arm's controllers.

:::warning
There also exists a `Robotiq URCap` maintained and provided by Robotiq. This URCap can be downloaded from the Robotiq support page and allows for a user to command the gripper through Polyscope.
While some users may prefer this configuration because it allows t#### URCaphem to write Polyscope programs that command the arm, this URCap will take absolute control over the gripper's communication and prevent the robot computer from commanding the gripper.
In other words, direct ROS control of the gripper will not be possible.
:::

#### Robotiq ROS 2 Driver
The Robotiq ROS 2 driver is not available through binaries, and therefore, must be built from source. The [`ros2_robotiq_gripper`](https://github.com/PickNikRobotics/ros2_robotiq_gripper) repository contains all the necessary Robotiq ROS 2 packages.
These are owned by PickNikRobotics and maintained with community support.

To build the driver, create a workspace if one does not exist already, and in that workspace clone [`ros2_robotiq_gripper`](https://github.com/PickNikRobotics/ros2_robotiq_gripper) repository, specifically choosing the `main` branch:
```bash
cd ~/colcon_ws/src
git clone https://github.com/PickNikRobotics/ros2_robotiq_gripper.git -b main
```

Then, the required dependencies must also be cloned from the `.repos` file:
```bash
cd ~/colcon_ws/src
vcs import --input ros2_robotiq_gripper/ros2_robotiq_gripper.rolling.repos
```

Now, the workspace can be built:
```bash
cd ~/colcon_ws/
colcon build --symlink-install --cmake-args=-DCMAKE_BUILD_TYPE=Release
```
