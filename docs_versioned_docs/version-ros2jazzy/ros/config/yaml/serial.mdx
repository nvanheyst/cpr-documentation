The Clearpath serial number is composed of two sections, separated by a hyphen, `-`, for example, `a300-00001`:
1. An alpha-numerical code corresponding to the robot platform, `a300`
2. Followed by an integer corresponding to the unit number, `00001`. For the A300, it is 5 digits and the rest of the platforms it is the last 4 digits.

At this moment, the supported robot platforms are:

1. [Husky A300](/docs_robots/outdoor_robots/husky/a300/user_manual_husky): `a300-00000`
2. [Husky A200](/docs_robots/outdoor_robots/husky/a200/user_manual_husky): `a200-0000`
3. [Dingo-D](/docs_robots/indoor_robots/dingo/user_manual_dingo): `dd100-0000` or `dd150-0000`
4. [Dingo-O](/docs_robots/indoor_robots/dingo/user_manual_dingo): `do100-0000` or `do150-0000`
5. [Jackal J100](/docs_robots/outdoor_robots/jackal/user_manual_jackal): `j100-0000`
6. [Ridgeback R100](/docs_robots/indoor_robots/ridgeback/user_manual_ridgeback): `r100-0000`
7. [Warthog W200](/docs_robots/outdoor_robots/warthog/user_manual_warthog): `w200-0000`

Every robot platform has specific attachments that are selected based on the serial number passed. Therefore, it is **required** that a serial number is specified in the `robot.yaml`.

## Sample

In our sample, we use a **Husky A300** and therefore have set our serial number to:

```yaml
serial_number: "a300-00000"
```
