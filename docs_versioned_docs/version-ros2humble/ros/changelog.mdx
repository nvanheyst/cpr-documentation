## [1.3] 2025-04-15

### New Features
- Added delay to manipulator controller
- Added Manipulator URDF Parameters
- Add UR socat tool communication script
- Add argument to disable generation in simulation

### Fixed
- Fixed how NMEA sentences are formed on J100 MCU
- Do not check battery status on D100 MCU
- Check init functions for errors on all MCUs
- Configure MCU service fix for all MCUs


## [1.2] 2025-03-12

### New Features
- Added more Manipulator samples and poses
- Added support for link materials
- Added MoveIt parameters and enable option
- Added support for Ouster OS1
- Added Franka Manipulator
- Added IMU filter to IMU entries

### Fixed
- Fixed Isolate Universal Robots driver and client library dependencies
- Fixed dependency on SRDF plugins
- Changed sensor package location
- Fixed Puma motor driver data bug and initialization
- Fixed IMU filter parameters path
- Fixed handling empty namespace

## [1.1] 2025-01-15

### New Features
- Added Ewellix Lift
- Added option to disable EKF
- Added support for HE2411 and HE2410 batteries

### Fixed
- Fixed Ridgeback R100 rear light colour

## [1.0] 2024-11-27

### New Features
- DO100 and DO150 Platform
- Added support for Ridgeback and Dingo to `clearpath_nav2_demos`
- Added support Axis cameras
- Added pointcloud support to OakD
- Updated CAN interface for Pumas
- Updated `clearpath_platform_msgs`
- Added `clearpath_motor_msgs`
- Removed `clearpath_platform` and `clearpath_hardware_interfaces` to `clearpath_robot`
- Added vcan service
- Move `battery_state` to `clearpath_hardware_interfaces`
- Add clearpath_motor_drivers, puma_motor_driver
- Play simulation automatically

### Fixed
- Jackal driving performance issues at high speeds
- Fixed issues with setting Domain ID on MCU robots

## [0.3.0] 2024-09-10
:::warning
The `StopStatus` message in `clearpath_msgs` has an additional field in the `0.3.0` release. This message type is used by the MicroROS enabled, robot MCU; therefore, the firmware must be updated on all robot platforms, except the A200. See, the [robot installation section](./installation/robot.mdx#firmware-update) for information on updating the firmware.
:::
